# Contributing to Rockgarden

Rockgarden is open source (MIT) but the maintainer asks that you discuss proposed changes before making pull requests.

## Bug Reports

Open an issue with:

- Description of the bug
- Expected behavior
- Steps to reproduce

## Feature Requests

Open an issue first. Describe the use case and why it matters. No feature PRs without prior discussion and approval.

## Pull Requests

**Bug fixes:** Accepted when fixing documented or intended behavior. Must reference an issue.

**Features:** Only after issue discussion and approval from the maintainer. Must reference an issue.

## When in Doubt

Open an issue and ask.

## Development Setup

See the README for development commands. The short version:

```bash
just install   # install dependencies
just test      # run tests
just ci        # lint + tests (run before submitting)
```
